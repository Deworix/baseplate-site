Install Baseplate on macOS

The current DMG is not yet signed with an Apple Developer ID or notarized. macOS will therefore block the first launch by default. These are Apple's per-app exception steps; you do not need to disable Gatekeeper.

Download the latest DMG Apple's official instructions

Apple Silicon or Intel?

Choose mac-arm64.dmg for Apple Silicon (M1, M2, M3, M4 and newer) or mac-x64.dmg for an Intel Mac.

Open the current unsigned release

  1. Open the DMG and drag Baseplate into Applications.
  2. Try to open Baseplate once, then dismiss the macOS warning.
  3. Open System Settings → Privacy & Security.
  4. Scroll to Security and click Open Anyway next to Baseplate.
  5. Confirm with Open. macOS remembers this exception for the app.

Do not disable Gatekeeper globally. Commands that disable quarantine or system-wide checks reduce protection for every download, not just Baseplate.

Why can't the warning simply be removed?

Apple only grants normal downloaded-app trust to software signed with a Developer ID certificate and notarized by Apple. That certificate requires the paid Apple Developer Program. An ad-hoc signature is free, but it does not identify the developer and does not remove Gatekeeper's warning.

What will change later?

The release workflow is ready to sign, notarize and staple both Mac builds as soon as valid Apple credentials are added. Until then this page states the limitation instead of hiding it.
